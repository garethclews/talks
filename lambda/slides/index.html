What is functional programming?
  • Trying not to sound glib: Programming using functions
  • Implementation of 'lambda calculus'
  • Trying to treat functions in programming like functions in mathematics
Why bother?
  • Reduces the possible types of bugs you experience
  • Reduces boilerplate
  • Code is super easy to move around
  • Code is clear and obvious
  • Chaining functions is equivalent to function composition
  • Declarative

Okay all this is great, but FOR WHY?!

Benefits of functional programming...

  1. Hugely cut down on the possible types of bugs in your program
  2. Much easier to move code around
  3. Super easy to test in statically typed languages (damn!)
  4. Chaining methods is really composition of functions, with no side effects this can become a tool for succinct and safe code
  5. You avoid having mutable data as a key for a hash map (looking at you Emily) show of hands, who has done this?
  6. The current state of programming is moving away from an OO paradigm to one where functional programming is king, even if this takes 3, 4 or 5 generations, that inertia is there so being ahead of the curve will make you super employable and able to do your work for a long time to come
  7. You get to write smart functions to operate on dumb data, how is this not desirable?
All of this lets me:
  1. write smart functions to operate on dumb data
  2. avoid interaction
Rules
  1. no loops
  2. no non-ternary if statements
  3. function definitions are single returns
  4. no side effects
  5. no assignments inside functions
  6. no lists
  7. functions may only have at most one argument
